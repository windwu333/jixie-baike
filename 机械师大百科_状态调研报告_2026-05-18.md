## 一、文章内容质量 [严重问题]

### 1.1 模板克隆 — 内容"空壳化"

前20篇文章全部来自 **机械制图与CAD** 类别，且核心正文完全相同：

| 文章标题 | 文件 | 正文内容 |
|---------|------|---------|
| 参数化设计 | cad-3d-k2.md | 制图标准 / 视图表达 / CAD软件应用 / 参考数据 |
| 参数化设计—应用详解 | cad-3d-k2-v.md | 同上（一字不差） |
| 装配约束 | cad-3d-k3.md | 同上 |
| 装配约束—应用详解 | cad-3d-k3-v.md | 同上 |
| 工程图生成 | cad-3d-k4.md | 同上 |
| 工程图生成—应用详解 | cad-3d-k4-v.md | 同上 |

所有这6篇文章描述的都是CAD绘图基础，而标题却是完全不同的主题。**内容与标题严重不匹配。**

### 1.2 模板主题 — 文章结构高度模板化

从所有2220篇文章中提取的第一节标题分布：

```
## 概述       -> 363篇
## 工艺原理   -> 327篇
## 设计原理   -> 311篇
## 基本概念   -> 244篇
## 技术原理   -> 226篇
## 标准体系概述 -> 206篇
## 基础理论   -> 199篇
## 制图标准   -> 168篇
```

**这些通用章节标题说明文章是使用内容模板批量生成的，不是针对各主题的原创内容。**

### 1.3 主题与分类严重错配

典型例子：

| 文章 | 所属分类 | 主题问题 |
|------|---------|---------|
| 5Why分析法 | 力学与强度分析 | 5Why是质量管理工具，正文却在讲应力应变和有限元 |
| 铝合金 | 工程材料/黑色金属 | 铝合金是有色金属，错放在黑色金属类别下 |
| 3D打印 | 制造工艺/铸造 | 正文讲通用制造工艺，不是3D打印 |

**168篇文章包含 "是连接设计和制造的桥梁" 这句描述语**，出现在完全不同主题的文章结尾。

### 1.4 术语注释中的垃圾文本

### 1.6 质量等级分布极不均衡

```
Start（初稿）: 1002篇 (45.1%)
Draft（草稿）:  899篇 (40.5%)
C级:           313篇 (14.1%)
```

**90.2% 的文章处于 Start 或 Draft 质量等级**，仅有 6 篇达到 B 级。

### 1.7 术语注释来源不可靠

- **1735/2220 篇文章**的术语标注"来源: seed"（seed是无意义的默认值）
- 其他来源如 `variant-v4`、`v6-automotive`、`subterm_extraction` 均非权威来源
- 术语定义内容空洞，如 "*实体建模*是机械工程领域的重要概念"（无实质定义）

### 1.8 英文/中英混合标题

- **165篇文章**为纯英文标题（如 `title: "feeler gauge"`, `title: "SAE"`）
- **247篇**为中英混合（如 `title: "galvanized钢"`, `title: "aisi/钢identification编号"`）

### 1.9 "参见"链接彻底损坏

**1868/2220篇文章 (84%)** 的 "参见" 链接包含中文字符，产生无效URL：

```
- [坐标机](/p/cad-3d-t0-v-参见-坐标机/)     ← URL包含中文
- [caliper](/p/cad-3d-t1-参见-caliper/)    ← 同上
```

这些链接在站内完全无法导航。

## 二、网站显示效果问题

### 2.1 Meta描述中泄漏Markdown语法

**在 public/ 目录下检查到 4491 个HTML页面** 的 meta description 中包含 Markdown 加粗符号 `**`：

```html
<meta name="description" content="**参数化设计**是表达设计思想和传递技术信息的重要工具...">
```

这显示在 Google 搜索结果中会有明显的星号问题。根源是 `hugo.toml` 中 `description` 包含 markdown 但没有使用 `plainify` 处理。

### 2.2 缺少静态资源目录

`static/` 目录完全为空，但 `public/` 中的HTML引用了多个favicon：

```html
<link rel="icon" href="/jixie-baike/favicon.ico">
<link rel="icon" type="image/png" sizes="16x16" href="/jixie-baike/favicon-16x16.png">
<link rel="apple-touch-icon" href="/jixie-baike/apple-touch-icon.png">
```

这些文件不存在于项目中（仅在 scripts/themes/list/static/ 内有一个 `favicon.ico`）。

### 2.3 public/ 目录中有大量旧版残留

- public/ 下有 **4700个目录**，其中 **2480个** 来自旧的构建（05-17版本）
- 这些旧版目录在 `sitemap.xml` 中没有包含，但仍然可通过链接访问

### 2.4 缺少国际化（i18n）支持

- `i18n/` 目录不存在
- 网站上只配置了 zh 语言，**缺少英文版本的双语支持结构**
- 虽然有 `-v.md`（应用详解）变体文件（1002个），但缺乏真正的多语言路由

### 2.5 导航面包屑链接路径问题

自定义 `single.html` 中的面包屑导航硬编码了 `/categories/` 路径，但没有考虑 `baseURL` 子目录前缀。

### 2.6 术语注释渲染为脚注

Markdown 脚注（`[^t1]`）语法在文章正文中被渲染为可点击数字链接，内容显示在文章底部。但由于术语内容有垃圾文本，读者点击后会看到不相关或荒谬的内容。

### 5.2 PaperMod 主题覆盖情况

主题的 `layouts/` 提供了完整的 PaperMod 模板集（46个文件），但**缺乏针对百科内容特性的定制**：
- 无文章质量徽章（quality grade badge）
- 无术语注释的特殊渲染
- 无参考来源的图标或分类显示
- 无限时阅读时间矫正（CJK语言需要）

### 5.3 搜索功能

- Fuse.js 搜索已配置
- 输出格式包含 JSON（支持搜索索引）
- `search.md` 存在，但搜索界面依赖于 PaperMod 的搜索页面
