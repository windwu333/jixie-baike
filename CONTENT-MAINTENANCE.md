# 机械师大百科 — 内容维护规范

## 质量阶梯（6级）

| 级别 | 颜色 | 标准 | 对应行动 |
|------|------|------|---------|
| Draft | 红色 | 仅基本定义，1-2段，不足独立 | 继续扩充 |
| Start | 橙色 | 初步发展，有基本信息但明显不完整 | 补充核心内容 |
| C | 黄色 | 内容可观但缺重要信息或结构不足 | 完善结构与引用 |
| B | 金色 | 基本完整，结构合理，引用充分 | 持续改进 |
| GA | 绿色 | 优良：结构清晰、引用可靠、覆盖全面 | 评审晋升 |
| FA | 深绿 | 特色：全维度审查通过，最佳内容 | 保持更新 |

质量等级标记在文章 frontmatter 的 `quality` 字段。

## 内容结构标准

每篇文章应包含:

1. **导言** — 首句定义式 (**{主题}**是{分类}的{定义})，≤4段
2. **章节** — 2-5个 `##` 级标题，每节首段为节摘要
3. **总结** — `## 总结`，概括全文要点
4. **参见** — `## 参见`，链接相关主题（有 related_terms 时自动生成）
5. **术语注释** — `### 术语注释与数据来源`，术语脚注列表
6. **参考资料** — `### 参考资料`，引用来源
7. **延伸阅读** — `### 延伸阅读`
8. **外部链接** — `### 外部链接`

## 禁止项

- ❌ 第二人称（你、你的、您）
- ❌ 反问句
- ❌ 时间敏感词（正朝着、近年来、最新发展、当前）
- ❌ 无引用的事实陈述
- ❌ 缺失 quality_level 字段

## 编辑流程

### 新增文章
1. 创建 `content/website/YYYY-MM-DD-slug.md`
2. 确保 frontmatter 含 quality 字段
3. 遵循内容结构标准
4. 运行 `python3 scripts/quality-gate.py --file <path>` 检查
5. Hugo build 验证通过

### 升级质量等级
1. Draft→Start: 加至少1个引用，补定义
2. Start→C: 加章节，补2+引用
3. C→B: 加数据引用，≥3章节
4. B→GA: 加标准引用，覆盖全面
5. GA→FA: 全维度审查

### 批量生成
```bash
# 1. 更新文章定义
python3 scripts/batch-article-defs.py

# 2. 重生成内容
python3 scripts/generate-content-v2.py

# 3. 合规扫描
python3 scripts/s2-compliance-scan.py

# 4. 质量门禁
python3 scripts/quality-gate.py

# 5. Hugo 验证
hugo
```

## 争议处理

1. 质量等级争议 → 参考 criteria 量化标准裁定
2. 分类归属争议 → 按最具体分类原则
3. 引用可靠性争议 → 优先采用 GB/ISO/ASTM 标准
4. 事实准确性争议 → 举证责任在添加方
